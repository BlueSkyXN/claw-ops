## 1. 功能概述

### 1.1 产品解决的问题

该产品是一个**多角色 AI 工作治理平台**，面向以下场景：

- 需要把自然语言工作请求转化为**受治理、可跟踪、可审计、可干预**的执行流程
- 需要让多个专门角色承担不同职责，而不是自由聊天式协作
- 需要把“方案制定、质量审查、任务派发、并行执行、结果回报”拆成强约束的阶段
- 需要一个统一控制台查看状态、介入流程、切换模型、管理扩展能力、观察会话运行情况

它强调的不是“让多个角色自己讨论”，而是：

- **强制质量关卡**
- **明确权限边界**
- **全过程观测**
- **可人工干预**
- **对外部运行时的治理与管理**

### 1.2 归一化角色模型

为避免实现绑定，本文统一使用以下功能角色称谓：

- **提交者**：发起工作请求的人或外部系统
- **收件角色**：接收外部消息，判断是闲聊还是工作请求，并做初步整理
- **规划角色**：把请求拆解为执行方案、子任务、优先级和验收目标
- **审议角色**：对方案做质量审核，决定放行或退回修改
- **派发角色**：将已放行任务分派给专业执行角色，并负责汇总
- **专业执行角色**：按领域执行子任务，可并行工作
- **简报角色**：负责周期性采集资讯并生成汇总简报
- **平台操作者**：通过控制台观察和干预系统的人

### 1.3 当前已确认的功能范围

当前从源码与测试可确认的功能包括：

1. **请求接入与分类**
   - 接收外部渠道消息或控制台模板请求
   - 将闲聊与工作请求区分开
   - 为合格工作请求创建受治理工作项

2. **受治理工作项生命周期**
   - 工作项创建
   - 分拣
   - 方案制定
   - 强制审议
   - 派发
   - 专业执行
   - 汇总/复审
   - 完成 / 阻塞 / 取消

3. **治理控制台**
   - 工作项看板
   - 活跃度与健康监控
   - 归档成果浏览
   - 角色总览
   - 模型配置
   - 扩展能力配置
   - 运行时会话观察
   - 模板化任务创建
   - 资讯简报订阅与查看

4. **运行时治理与容错**
   - 自动派发
   - 停滞检测
   - 自动重试
   - 升级协调
   - 自动回滚到稳定节点
   - 服务重启后的待派发恢复

5. **扩展与运行时集成**
   - 角色目录与运行状态查看
   - 每角色模型切换
   - 本地扩展创建
   - 远程扩展拉取 / 更新 / 删除
   - 外部运行时会话映射为控制台可见对象

6. **运维与审计**
   - 健康检查
   - 活动流 / 事件流
   - 任务详情、时间线、资源消耗、进展记录、子任务清单

## 2. 接口契约（对每个关键功能）

### 2.1 工作请求接入与分类

#### 输入

- 自然语言请求文本
- 来源渠道元信息（可选）
- 发起方式：
  - 外部消息渠道
  - 控制台模板生成
  - 控制台直接创建

#### 输出

二选一：

1. **非工作型输入**
   - 直接生成回复
   - 不创建受治理工作项

2. **工作型输入**
   - 创建一个新的受治理工作项
   - 返回工作项标识
   - 返回已进入收件阶段的确认信息

#### 状态变化

- 非工作型输入：不进入工作项状态机
- 工作型输入：
  - 初始化时间线
  - 初始化调度元数据
  - 进入“收件处理”阶段
  - 自动向收件角色派发

#### 错误处理

- 空标题 / 空请求：拒绝创建
- 过短文本：拒绝创建
- 明显是闲聊或无效词：拒绝创建
- 明显是路径、元数据、噪声文本：先清洗；清洗后仍无效则拒绝
- 已完成或已取消的同标识工作项不得被覆盖

### 2.2 模板化任务创建

#### 输入

- 模板类型
- 模板参数字典
- 可选建议执行领域
- 优先级

当前已确认的模板类别包括：

- 周报
- 代码审查
- API 设计
- 竞品分析
- 数据分析报告
- 博客文章
- 部署方案
- 邮件/通知文案
- 站会摘要

#### 输出

- 渲染后的工作请求预览
- 发起后的工作项标识
- 创建成功/失败结果

#### 状态变化

- 成功提交后，与普通工作请求进入同一治理状态机
- 记录模板标识和参数快照
- 可记录建议执行领域，作为派发参考而非强制约束

#### 错误处理

- 缺少必填模板参数：拒绝提交
- 控制面不可用：返回创建失败
- 外部运行时未就绪：应给出风险提示；是否允许继续提交由平台策略决定

### 2.3 受治理工作项查询

#### 输入

- 过滤条件：
  - 状态
  - 执行领域
  - 优先级
  - 是否归档
  - 文本搜索 `[待确认：全文搜索范围是否固定]`
- 分页参数

#### 输出

- 工作项列表
- 总数或当前页数量
- 每个工作项的摘要信息：
  - 标识
  - 标题
  - 当前阶段
  - 当前负责角色/领域
  - 当前进展摘要
  - 阻塞原因
  - 更新时间
  - 归档状态

#### 状态变化

- 只读，不改变状态

#### 错误处理

- 非法过滤值：返回参数错误
- 分页超限：返回边界错误或自动裁剪

### 2.4 工作项生命周期控制

该功能覆盖：

- 审议放行
- 审议退回
- 手动推进到下一阶段
- 叫停
- 取消
- 恢复
- 归档 / 取消归档

#### 输入

- 工作项标识
- 操作类型
- 可选评论 / 理由

#### 输出

- 操作是否成功
- 新阶段或新状态
- 是否已触发下一角色派发
- 人类可读的结果消息

#### 状态变化

- **审议放行**
  - 若处于方案审议阶段：转入派发准备阶段
  - 若处于最终复审阶段：转入完成阶段
- **审议退回**
  - 转回规划阶段
  - 审议轮次计数递增
- **手动推进**
  - 按顺序推进到下一阶段
  - 自动派发下一角色
- **叫停**
  - 转入阻塞状态
  - 记录前一状态，便于恢复
- **取消**
  - 转入取消状态
  - 记录前一状态
- **恢复**
  - 回到暂停前状态，或默认回到执行状态
  - 如非终态则重新派发
- **归档**
  - 标记为归档
  - 支持单条归档与批量归档已结束项

#### 错误处理

- 工作项不存在：返回不存在错误
- 当前状态不允许该操作：返回状态错误
- 未知操作：返回参数错误
- 未提供必填工作项标识：返回参数错误

### 2.5 进度与子任务清单上报

#### 输入

- 工作项标识
- 当前进度一句话描述
- 可选子任务清单
- 可选资源消耗信息：
  - token 数
  - 成本
  - 耗时
- 可选某一子任务的详细说明

#### 输出

- 成功/失败结果
- 更新后的完成度摘要

#### 状态变化

- 更新工作项当前进展摘要
- 追加一条进度日志
- 可替换或更新子任务清单
- 更新最后活跃时间
- 不强制改变生命周期阶段

#### 错误处理

- 工作项不存在：返回不存在错误
- 非法子任务状态：自动回落到默认状态或返回参数错误
- 资源字段格式非法：忽略非法字段或归零处理

### 2.6 自动派发与恢复机制

#### 输入

- 工作项标识
- 当前阶段
- 可选触发原因：
  - 新建
  - 状态切换
  - 人工推进
  - 自动重试
  - 自动回滚
  - 启动恢复

#### 输出

- 派发入队结果
- 派发执行结果：
  - 成功
  - 失败
  - 超时
  - 网关不可用
  - 异常

#### 状态变化

- 记录最近派发时间
- 记录目标角色
- 记录派发触发原因
- 记录最近派发状态
- 在时间线中追加派发记录

#### 错误处理

- 当前阶段无对应角色：跳过自动派发
- 外部运行时不可用：记录“不可用”状态
- 派发失败/超时：保留失败原因供人工介入

### 2.7 停滞检测、重试、升级与回滚

#### 输入

- 扫描阈值（默认约 180 秒）
- 工作项最近活跃时间
- 当前调度元数据

#### 输出

- 本轮扫描结果
- 动作列表：
  - 重试
  - 升级协调
  - 回滚

#### 状态变化

- 对超时未活跃且非终态、非归档、非阻塞的工作项执行检查
- 首先执行自动重试（有次数上限）
- 达到上限后升级给更高治理角色协调
- 若仍持续停滞并允许自动回滚，则回退到最近稳定节点
- 记录升级层级、重试次数、停滞起点和时间线

#### 错误处理

- 扫描参数过小：自动提升到安全最小值
- 工作项状态无效：跳过
- 无稳定快照可回滚：只保留升级结果，不执行回滚

### 2.8 活动流、审计流与详情视图

#### 输入

- 工作项标识
- 可选过滤条件：
  - 事件主题
  - 生产者
  - 追踪标识
  - 数量上限

#### 输出

- 活动流：
  - 生命周期流转
  - 进度上报
  - 资源消耗
  - 工具/消息摘要
  - 子任务变化
- 事件流：
  - 事件主题
  - 类型
  - 生产者
  - 时间戳
  - 负载

#### 状态变化

- 只读查询，不改变状态

#### 错误处理

- 工作项不存在：返回不存在错误
- 过滤参数非法：返回参数错误

### 2.9 角色目录、状态监控与模型配置

#### 输入

- 查询全部角色或单个角色
- 可选角色标识
- 可选目标模型
- 可选唤醒消息

#### 输出

- 角色目录：
  - 标识
  - 展示名
  - 职责
  - 图标
  - 允许协作对象
  - 当前模型
  - 已装扩展
- 角色状态：
  - 运行中 / 空闲 / 离线 / 未配置
  - 最后活跃时间
  - 会话数
  - 网关健康状态
- 模型变更结果
- 唤醒结果

#### 状态变化

- 模型变更请求进入待应用队列
- 应用后应触发运行时重载或重启
- 唤醒操作向目标角色投递提示消息

#### 错误处理

- 角色不存在：返回不存在错误
- 模型参数缺失：返回参数错误
- 外部运行时未就绪：返回执行失败

### 2.10 扩展能力管理

该功能覆盖：

- 创建本地扩展
- 查看扩展内容
- 从远程或本地源引入扩展
- 列出远程管理扩展
- 更新远程管理扩展
- 删除远程管理扩展

#### 输入

- 角色标识
- 扩展名称
- 描述
- 可选触发条件
- 可选远程源地址或本地文件路径

#### 输出

- 创建/导入/更新/删除结果
- 扩展元信息：
  - 来源
  - 本地路径
  - 添加时间
  - 最近更新时间
  - 校验状态

#### 状态变化

- 本地扩展创建后，角色配置刷新
- 远程扩展引入后保存来源元数据
- 更新时重新抓取并替换本地内容
- 删除时移除受控扩展目录

#### 错误处理

- 名称非法：拒绝
- 角色不存在：拒绝
- 远程地址不安全：拒绝
- 本地路径越界：拒绝
- 文件格式不符合要求：拒绝
- 试图删除非远程受控扩展：拒绝

### 2.11 运行时会话观察

系统除受治理工作项外，还观察外部运行时的普通会话与子任务执行会话。

#### 输入

- 外部运行时会话清单
- 会话日志
- 过滤条件：
  - 全部
  - 活跃
  - 按角色

#### 输出

- 会话卡片列表：
  - 会话标识
  - 角色来源
  - 渠道标签
  - 活跃状态
  - 最近消息摘要
  - 资源消耗
  - 最近更新时间
- 会话详情：
  - 最近活动序列
  - 输入/输出 token
  - 日志摘要

#### 状态变化

- 会话对象为**观测性对象**
- 不进入完整治理工作流
- 根据外部会话活跃时间自动映射到“执行中 / 汇总中 / 待处理 / 阻塞”等观测状态

#### 错误处理

- 外部会话目录不存在：返回空列表
- 日志损坏：跳过损坏记录，不中断整体展示

### 2.12 资讯简报

#### 输入

- 分类启用配置
- 用户自定义关键词
- 自定义资讯源
- 可选企业消息推送地址
- 手动刷新请求

#### 输出

- 分类后的资讯列表
- 每条资讯包含：
  - 标题
  - 摘要
  - 原始链接
  - 来源
  - 发布时间
  - 图片（可选）
- 推送结果

#### 状态变化

- 定时采集或手动触发采集
- 写入“最新简报”
- 写入按日期归档的历史简报
- 成功采集后才写入幂等锁
- 如配置了推送目标，可将当日简报推送到企业消息渠道

#### 错误处理

- 资讯源不可访问：跳过该源
- 自定义源地址不安全：拒绝纳入
- 推送地址非法：拒绝保存或拒绝推送
- 采集失败：不应写入成功锁

### 2.13 平台健康与诊断

#### 错误处理

- 依赖不可用：返回降级状态而不是误报成功

- **错误处理**：重复审批、超时审批、无权限审批

#### 2.14.2 `[待确认]` 角色绩效体系

- **输入**：时间范围、角色范围
- **输出**：完成率、返工率、耗时、资源消耗、排行榜、趋势
- **状态变化**：统计类功能，原则上只读
- **错误处理**：统计周期为空、缺少基础数据

#### 2.14.3 `[待确认]` 消息流可视化

- **输入**：事件订阅、时间范围、筛选条件
- **输出**：角色间流向、消息类型、时间线回放
- **状态变化**：只读
- **错误处理**：事件缺失或无法重建链路时需明确提示

#### 2.14.4 `[待确认]` 知识沉淀与相似任务检索

- **输入**：新请求或查询词
- **输出**：相似历史工作项、引用链、相关经验
- **状态变化**：可新增索引或知识条目
- **错误处理**：索引为空、引用链断裂、相关度不足


## 3. 行为规则

### 3.1 工作项生命周期规则

已确认的顺序性阶段为：

1. 待处理
2. 收件处理
3. 方案制定
4. 审议
5. 派发准备
6. 待执行 / 执行中
7. 汇总 / 复审
8. 完成

并存在附加状态：

- 阻塞
- 取消

规则：

- 所有工作请求在执行前必须经过审议阶段
- 审议可以放行，也可以退回规划阶段
- 专业执行可并行发生，但汇总结果应重新回到治理角色
- 进度上报不等于阶段变更
- 手动推进只允许按顺序前移

### 3.2 强制审核规则

- 方案不允许直接进入执行
- 审议结果必须是“放行”或“退回”
- 审议退回会保留轮次记录
- 审议通过后才允许进入派发
- 最终完成前还存在一次结果汇总/确认阶段

关于审议最多可退回多少轮，仓库叙事中有多轮示例，但代码中未发现统一硬上限，因此标记为 `[待确认]`。

### 3.3 自动派发规则

- 新工作项创建后自动派发给收件角色
- 每次合法阶段切换后，如该阶段存在责任角色，应自动派发
- 派发失败不应静默吞掉，必须留下状态和错误摘要
- 平台重启后应重新尝试恢复“入队但未完成”的派发

### 3.4 停滞处理规则

- 默认停滞阈值约为 180 秒
- 非终态、非归档、非阻塞工作项才参与停滞扫描
- 处理顺序：
  1. 自动重试
  2. 升级协调
  3. 自动回滚到稳定节点

自动回滚前提：

- 存在稳定快照
- 当前策略允许自动回滚

### 3.5 人工干预规则

- 平台操作者可以对工作项执行：
  - 叫停
  - 取消
  - 恢复
  - 审批
  - 手动推进
  - 归档

干预规则：

- 恢复应优先回到暂停前状态
- 已完成或已取消项不应被自动重新派发
- 归档项不再参与自动扫描

### 3.6 进度与子任务规则

- 任何角色都可追加阶段内进度说明
- 进度说明应简洁描述“当前正在做什么”
- 子任务清单支持三态：
  - 未开始
  - 进行中
  - 已完成
- 子任务详细说明可承载阶段性产出
- 进度日志应限制长度，避免无限增长

### 3.7 输入清洗与工作请求有效性规则

创建工作项前需执行以下处理：

- 去掉对话元数据
- 去掉代码块或结构化残余
- 去掉明显文件路径
- 去掉 URL
- 去掉常见前缀
- 压缩多余空白
- 超长内容截断

若清洗后内容仍满足以下任一条件，应拒绝创建：

- 太短
- 只有标点
- 属于常见无效短语
- 看起来仍像文件路径
- 清洗后为空

### 3.8 模板与建议执行领域规则

- 模板只负责“规范化请求生成”
- 模板可携带建议执行领域、时间估计和成本估计
- 建议执行领域不是最终派发结果的硬约束
- 模板参数变更应先可预览，再可提交

### 3.9 运行时会话观察规则

- 运行时会话与受治理工作项属于不同对象
- 会话观察以“展示状态、摘要、活动、资源消耗”为主
- 会话卡片不应被误当作完整治理工作项推进
- 会话标题可根据来源和会话类型进行人类友好重命名

### 3.10 简报规则

- 简报按分类抓取外部资讯源
- 应去重、按关键词做筛选或加权
- 同一天内应避免高频重复生成
- 手动刷新可绕过当天幂等限制
- 成功采集后更新“最新简报”并归档到按日快照

### 3.11 未来目标规则（均为 `[待确认]`）

- `[待确认]` 人工审批是否发生在方案审议后、派发前，还是替代部分审议动作
- `[待确认]` 绩效评分的权重模型
- `[待确认]` 知识检索的召回范围与排序逻辑
- `[待确认]` 实时消息流视图的保留时长、过滤粒度与回放能力


## 4. 非功能性约束

### 4.1 可观测性

系统必须做到：

- 每个工作项可见当前阶段
- 每次状态变化可审计
- 每次人工动作有记录
- 每次派发与失败可追踪
- 每条进度与子任务变化可回看
- 角色在线状态与最近活跃时间可见
- 运行时会话活动可见

### 4.2 响应与更新时效

当前已确认的体验约束包括：

- 控制台数据刷新以“近实时”为目标
- 当前主运行模式存在大约 **15 秒** 级别的周期性刷新链路
- 模型切换后运行时应在大约 **5 秒** 左右完成生效
- 手动触发资讯刷新后，控制台预计在 **30–60 秒** 量级看到结果
- 自动停滞扫描默认按 **180 秒** 判断

未来目标为：

- `[待确认]` 改为 WebSocket 级事件推送，而非以轮询为主

### 4.3 可靠性

系统应满足：

- 控制面数据更新具备原子性，避免并发写坏
- 中断后可恢复未完成派发
- 停滞任务不会无限沉默
- 关键依赖不可用时返回降级状态而非误报成功
- 终态与归档项不被自动流程重复扰动

### 4.4 并发与协作

- 专业执行角色允许并行工作
- 多角色可同时向同一工作项写入进度
- 并发写入下，子任务与进度信息不应破坏数据结构
- 自动扫描、手动干预与外部运行时更新可能并发发生，平台必须保证结果可解释

### 4.5 兼容性

当前产品依赖一个外部多角色运行时，外部运行时至少需要具备：

- 多角色定义
- 每角色独立工作空间
- 可控消息渠道
- 可读取会话清单与会话日志
- 可切换模型
- 可发起角色级唤醒或派发

未来目标兼容性：

- `[待确认]` 增加数据库、事件总线、WebSocket 客户端、容器编排环境

### 4.6 安全与权限

系统当前可确认的安全边界包括：

- 角色间协作有明确白名单
- 扩展名称与角色标识只允许安全字符
- 远程扩展来源只允许安全协议；本地路径需限制在允许根目录内
- 企业消息推送地址应使用 HTTPS 且受域名白名单约束
- 明显危险或越界路径必须拒绝
- 需要人工确认的破坏性动作应有显式理由记录

### 4.7 数据保留与审计

- 工作项时间线应保留
- 进度与子任务变化应保留
- 归档后成果仍可检索
- 会话观察数据与工作项数据应区分来源


## 5. 边界条件与已知陷阱

### 5.1 输入边界

- 过短、纯噪声、路径型文本可能被误当工作请求；当前策略倾向于严格拒绝
- 标题清洗和备注清洗并非完全相同，长度上限不同
- 路径、URL、对话元数据混入请求时，可能导致有效内容被过度截断 `[待确认：是否需要可解释的清洗预览]`

### 5.2 生命周期边界

- 审议动作只在特定阶段有效
- 手动推进只适用于顺序阶段，不适用于任意跳转
- 终态或归档项不会参与自动重试/自动升级/自动回滚
- 阻塞项当前默认不参与自动重试

### 5.3 标识体系边界

- 当前可见工作项标识是人工友好的日期+序号格式
- 进行中的未来架构又引入了更偏系统内部的持久标识和追踪标识
- 因此“唯一对外主标识”在长期形态上仍属 `[待确认]`

### 5.4 观察对象混用风险

- 平台同时展示“受治理工作项”和“运行时会话卡片”
- 两者外观相似，但语义不同
- 如果未来重构控制台，必须避免把会话对象误用为完整工作项对象

### 5.5 当前与未来控制面的分裂

从仓库可见，产品目前同时存在：

- 一个以文件和本地控制台为中心的当前主运行面
- 一个以持久存储、事件总线和推送为目标的未来运行面

因此以下问题都应视为 `[待确认]`：

- 长期主控制面到底是哪一套
- 外部 API 的最终统一形状
- 前端未来将绑定哪套控制面

### 5.6 审议与回退策略边界

- 审议退回轮次数会累计
- 但仓库没有给出统一、硬性的最大退回轮次规则
- 这意味着“无限退回”还是“超过阈值后强制放行/终止”属于 `[待确认]`

### 5.7 停滞恢复边界

- 自动重试次数、升级层级和回滚节点依赖调度元数据
- 若快照缺失或快照本身异常，自动回滚可能无法执行
- 回滚后重新派发是否一定成功，取决于外部运行时可用性

### 5.8 扩展能力边界

- 本地创建扩展与远程受控扩展的生命周期不同
- 远程扩展删除接口不会删除本地手工扩展
- 外部扩展内容当前主要校验“结构像不像扩展”，并非完整语义校验

### 5.9 资讯简报边界

- 资讯抓取依赖第三方源稳定性
- 分类命中依赖关键词匹配，可能出现误判或漏判
- 同日幂等锁会防止重复生成，但失败时应避免误写锁文件

### 5.10 未来功能的确认边界

以下功能虽然在公开路线图中明确存在，但若要作为开发依据，必须在实施前补足更严格 spec：

- `[待确认]` 人工审批的权限与通知链路
- `[待确认]` 绩效评分公式与数据来源
- `[待确认]` 知识检索的数据保鲜策略与引用可信度
- `[待确认]` 消息流可视化的隐私边界与保留策略
- `[待确认]` 第三方适配器的同步方向、冲突解决和审计要求
