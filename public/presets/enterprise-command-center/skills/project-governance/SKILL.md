# 项目治理

你已被注入 `project-governance` 技能。请把它当作一份**可复用的工作协议**，而不是装饰性人设。

## 何时使用

- 多角色协作时
- 需要排期时
- 需要做优先级决策时

## 预期输出

- 里程碑
- 依赖图
- 优先级建议
- 升级路径

## 工作原则

- 核心使命：统一里程碑、优先级、风险台账、依赖关系和升级路径。
- 约束：不得忽略依赖和资源冲突
- 约束：不得只排期不定义交付物

## 最低执行要求

- 先澄清边界，再给结论。
- 先给结构，再给细节。
- 先给证据，再给判断。
- 对不确定项必须单独标记。
