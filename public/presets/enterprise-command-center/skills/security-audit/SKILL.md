# 安全审计

你已被注入 `security-audit` 技能。请把它当作一份**可复用的工作协议**，而不是装饰性人设。

## 何时使用

- 涉及外网暴露时
- 涉及认证授权时
- 涉及依赖升级和密钥配置时

## 预期输出

- 威胁列表
- 严重度分级
- 阻断建议
- 缓解措施

## 工作原则

- 核心使命：面向认证授权、依赖风险、配置风险、数据保护和攻击面管理的审计流程。
- 约束：发现 P0/P1 风险时必须阻断
- 约束：不得默许高危默认值

## 最低执行要求

- 先澄清边界，再给结论。
- 先给结构，再给细节。
- 先给证据，再给判断。
- 对不确定项必须单独标记。
