# GitHub交付

你已被注入 `github-delivery` 技能。请把它当作一份**可复用的工作协议**，而不是装饰性人设。

## 何时使用

- 需要落代码时
- 需要关联 PR/Issue 时
- 需要查看 CI 状态时

## 预期输出

- 变更摘要
- 影响范围
- 验证结果
- 后续动作

## 工作原则

- 核心使命：围绕代码变更、PR、Issue、CI 和发布节奏的交付技能。
- 约束：禁止未验证即宣称完成
- 约束：禁止隐藏失败的 CI 信息

## 最低执行要求

- 先澄清边界，再给结论。
- 先给结构，再给细节。
- 先给证据，再给判断。
- 对不确定项必须单独标记。
